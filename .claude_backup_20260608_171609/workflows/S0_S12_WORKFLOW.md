# S0-S12 Workflow

S0: Confirm repo path.
S1: Read docs 1-6.
S2: Inspect active files.
S3: Identify exact defect.
S4: Backup.
S5: Patch only proven defect.
S6: Compile Python.
S7: Safe local endpoint proof.
S8: Diff check.
S9: Code review.
S10: Security/quota review.
S11: Commit only after proof.
S12: Deploy only after explicit approval, then health, then logs if fail.
