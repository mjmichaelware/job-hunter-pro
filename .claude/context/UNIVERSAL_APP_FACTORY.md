# Universal App Factory Workflow

Use this to create or repair any app.

## Phase 1 — Intent

- Identify owner.
- Identify product name.
- Identify users.
- Identify core workflow.
- Identify data sources.
- Identify outputs.
- Identify safety constraints.

## Phase 2 — Architecture

- Define frontend.
- Define backend.
- Define persistence.
- Define providers.
- Define auth.
- Define observability.
- Define deploy target.

## Phase 3 — Contracts

- API contracts.
- UI contracts.
- provider contracts.
- storage contracts.
- security contracts.

## Phase 4 — Implementation

- patch only confirmed defects
- keep modules decoupled
- keep provider adapters replaceable
- keep UI rendering honest
- keep proof repeatable

## Phase 5 — Proof and handoff

- compile
- safe proof
- route proof
- UI proof
- diff check
- proof report
- commit
- export handoff
