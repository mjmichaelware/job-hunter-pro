# Omega Alpha Standard — AI Workflow OS

This is the master standard for reusable AI-assisted software creation.

## Core principle

Do not let an AI agent guess, drift, fake proof, burn quota, hide uncertainty, or patch blindly.

The workflow must preserve:

- product intent
- current code truth
- contradiction history
- bug evidence
- patch plans
- proof results
- handoff state

## Universal build loop

1. Read state.
2. Inspect current files.
3. Reconcile specs.
4. Map routes, data, UI, storage, security, providers, and tests.
5. Record defects with evidence.
6. Write patch plan.
7. Patch surgically.
8. Compile.
9. Run safe proof.
10. Run diff checks.
11. Write proof report.
12. Commit only proven work.
13. Push only when authorized.
14. Deploy only when authorized.
15. Export handoff packet.

## Never claim perfection

No workflow is perfect. The system must state uncertainty and require proof.
