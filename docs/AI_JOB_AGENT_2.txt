================================================================================
KEY DECOUPLING RULES
================================================================================
1. api/ files contain ZERO business logic вЂ” they call pipeline/, search/, store/.
2. Each provider is one file, behind providers/base.py ABC. Adding a source = one new file + registry line.
3. Each industry is one file, behind industries/base.py. Adding an industry = one new file.
4. LLMs live in providers/reasoning/ and can ONLY classify/extract/score over text we pass them.
   They are physically separated from providers/search/ so they can never be wired as discovery. No hallucinated jobs.
5. store/ is the only layer that talks to Firestore. Swap to Postgres later = change store/ only.
6. budget.py is consulted before every paid call; cache_repo checked before every external fetch.
7. Nothing hardcodes a secret. Providers read keys at runtime and go dormant if absent.
